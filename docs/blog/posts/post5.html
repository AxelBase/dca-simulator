Filtering & Exporting Long Simulations (520+ Periods)

Published: January 06, 2026

The DCA Simulator excels at handling extended investment horizons, supporting up to one thousand periods while maintaining sub-second performance. The interactive results table makes exploring long-term weekly or monthly plans intuitive and productive.

Live Period Search

Type any number in the search field and the table instantly filters to matching periods. Partial matching allows typing twenty to see periods twenty, one hundred twenty, two hundred twenty, and so on. This is perfect for jumping to specific months or years in decade-long simulations.

Table Controls

  • Search field with clear button for quick reset
  • Visibility toggle to hide the table when only summary metrics are needed
  • Live badge showing filtered versus total period count
  • Responsive design that works smoothly on mobile devices

CSV Export Feature

One click downloads a complete file containing both the summary metrics and every individual period row. The export includes headers and clean formatting, ready for import into spreadsheets or data tools. This enables offline charting, custom calculations, or archiving of specific scenarios.

Performance in Long Runs

Even at five hundred twenty weekly periods—representing ten full years—the calculation and table rendering complete almost instantly. The lightweight JavaScript core ensures smooth experience without pagination or loading delays. Search filtering remains responsive regardless of dataset size.

Practical Use Cases

Long simulations are valuable for retirement planning projections, comparing decade-long weekly versus monthly strategies, or stress-testing extreme growth assumptions. Exporting data allows combining multiple runs or building personal dashboards outside the browser.

FAQ

How many periods can I simulate?

Up to one thousand periods with full performance—equivalent to nearly twenty years monthly or ten years weekly.

Does filtering affect the summary?

No—the summary always reflects the complete simulation; filtering only changes table visibility.

What format is the CSV file?

Standard comma-separated values with summary rows followed by detailed period data.
